# 技术调研记录目录

本目录用于保存各模块在开发前的调研文档，统一格式如下：

```
# <模块名称> 调研报告

## 1. 需求概述
- 业务场景与问题陈述
- 性能 / 安全 / 合规约束
- NAS 部署与集成要求

## 2. 候选方案对比
| 名称 | 类型/许可证 | 部署方式(NAS/云) | 主要优劣 | 维护成本 | 备注 |

## 3. 实验 / POC
- 环境与步骤
- 关键日志/截图
- 资源占用（CPU/GPU/内存/磁盘）

## 4. 集成评估
- 与 SmartDoc 现有模块的接口点
- 所需适配/SDK
- 风险与缓解措施

## 5. 推荐结论
- 首选方案 + 理由
- 备选方案 + 启用条件
- TODO / 后续工作
```

> 所有调研文档需在开始编码前提交给 Coordinator Agent 审批，并在对应 PR 中引用。
